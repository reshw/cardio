# 줄넘기 마일리지 모델 검토

## 배경

현재 클럽 마일리지는 단일 단위 기반이다.
- 달리기: Xkm → 마일리지
- 계단: X층 → 마일리지
- 시간제: X분 → 마일리지

줄넘기는 **시간(분)** + **개수(개)** 두 가지 변수를 동시에 가져서 기존 방식으로는 처리 불가.

---

## 기준 조건

> **5분 동안 500개 = 1 마일리지**

- 기준 페이스: **100개/분**
- 두 조건(시간 AND 개수) 모두 충족해야 의미 있음

---

## 검토 모델

### 옵션 A — 페이스 비율 방식 (권장 후보)

```
마일리지 = 개수 ÷ (시간(분) × 100)
```

| 사례 | 계산 | 결과 | 해석 |
|---|---|---|---|
| 5분 + 500개 | 500 ÷ (5×100) | **1.0** | 기준 |
| 4분 + 500개 | 500 ÷ (4×100) | **1.25** | 빠른 페이스 → 보상 |
| 10분 + 500개 | 500 ÷ (10×100) | **0.5** | 느린 페이스 → 감점 |
| 5분 + 1000개 | 1000 ÷ (5×100) | **2.0** | 많이 할수록 비례 증가 |

**장점:** 직관적, 빠르게 많이 하면 높은 마일리지
**단점:** 쉬엄쉬엄 오래 하면 페이스가 낮아져 불이익

---

### 옵션 B — 개수 기준 + 페이스 최소 요건

```
페이스 ≥ 100개/분 이면 → 마일리지 = 개수 ÷ 500
페이스 < 100개/분 이면 → 마일리지 = 0 (또는 비율 감점)
```

| 사례 | 페이스 | 결과 | 해석 |
|---|---|---|---|
| 4분 + 500개 | 125개/분 ✓ | **1.0** | 기준 이상이면 개수만 인정 |
| 10분 + 500개 | 50개/분 ✗ | **0** 또는 **0.5** | 페이스 미달 |
| 5분 + 1000개 | 200개/분 ✓ | **2.0** | 개수 비례 |

**장점:** 단순함, "최소 페이스만 넘으면 개수대로"
**단점:** 기준을 살짝 못 맞추면 0이 되는 절벽 효과 있음

---

## 결정해야 할 것

### 1. "4분 + 500개" 를 어떻게 볼 것인가?

- **더 좋은 것이다** (빠른 페이스는 보상) → 옵션 A
- **5분 기준을 넘겼으니 동일하다** → 옵션 B

### 2. "10분 + 500개" 를 어떻게 볼 것인가?

- **느린 거니까 패널티** → 옵션 A
- **어쨌든 500개 했으니 인정 (단 감점 없음)** → 옵션 B 변형

---

## 구현 관점 메모

- 줄넘기 운동 기록에 **시간(분)** 과 **개수(개)** 두 필드가 모두 저장되어야 함
- 현재 `mileage_config` 는 단일 계수(`number`) 구조 → 줄넘기는 별도 타입 필요할 수 있음
- 또는 줄넘기를 `분` 단위로만 받고, 개수는 메모 처리하는 우회책도 있음 (단순하지만 페이스 반영 불가)
