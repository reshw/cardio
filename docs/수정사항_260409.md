# 수정사항 260409

## 1. WorkoutDetail 수정 실패 오류 수정

**문제**
- `/workout/{uuid}` 페이지에서 운동 기록 수정 시 실제로는 성공했는데 "수정에 실패했습니다" 알럿이 뜨는 문제

**원인**
- `workoutService.updateWorkout()`에서 workout 업데이트 자체는 성공했으나, 이후 클럽 마일리지 스냅샷 재생성 단계에서 에러 발생 시 throw가 위로 전파됨
- `club_members` 테이블에 존재하지 않는 `status` 컬럼으로 필터링하던 쿼리가 400 오류 유발

**수정**
- `updateWorkout` 내 마일리지 스냅샷 재생성 블록을 try-catch로 감싸 실패해도 수정은 성공 처리 (`createWorkout`과 동일한 패턴으로 통일)
- `.eq('status', 'active')` 제거

## 2. WorkoutDetail 수정 폼 날짜 KST 보정

**문제**
- 수정 모드 진입 시 날짜/시간 input에 UTC 기준 시간이 표시됨 (KST보다 9시간 빠름)
- 저장 시 브라우저 timezone에 따라 DB 저장값이 달라질 수 있음

**수정**
- 표시: UTC → KST(+9h) 보정 후 input에 세팅
- 저장: `workoutTime + ':00+09:00'`으로 KST 명시하여 UTC 변환 후 DB 저장
- 취소/재조회 시에도 동일하게 KST 보정 적용

## 3. ClubMemberDetail URL 개인정보 노출 차단

**문제**
- 마일리지 탭에서 멤버 카드 클릭 시 `/club/member/{clubId}/{userId}/{닉네임}` 구조로 URL에 userId와 닉네임이 그대로 노출
- 순차 탐색(IDOR)으로 타인의 이름·기록 조회 가능

**수정**
- 라우트를 `/club/member/:clubId`로 변경 (userId, userName 제거)
- userId·userName은 `navigate state`로 전달 (URL에 미노출)
- state 없이 직접 URL 접근 시 `/club`으로 리다이렉트

**변경 파일**
- `src/App.tsx` — 라우트 경로 수정
- `src/pages/Club.tsx` — navigate 시 state로 전달
- `src/pages/ClubMemberDetail.tsx` — useParams 대신 location.state 사용
