---

## 2. 왜 "기록실"이 아닌 "클럽" 기능인가

- 기록실 개인 목표 = 혼자만 앎 → 심리적 구속력 없음
- 클럽 챌린지 = **공개 선언** → 클럽원들이 내 목표를 앎 → 달성률 올라감
- 클럽장의 역할: 분위기 메이커. 챌린지를 여는 행위 자체가 클럽 운영.

---

## 3. 확정된 설계

### 위계 구조

```
[클럽이 여는 것 — 판(場)]
"26년 05월 연휴 챌린지!"  5/1 ~ 5/7

  └─ 홍길동의 "26년 05월 연휴 챌린지!"  →  러닝 30km 목표
  └─ 김철수의 "26년 05월 연휴 챌린지!"  →  수영 5km 목표
  └─ 이영희의 "26년 05월 연휴 챌린지!"  →  사이클 100km 목표
```

- `challenges` = 판 (제목 + 기간)
- `challenge_participants` = 개인 선언 (종목 + 목표값)
- 챌린지는 기존 `workouts` 기록을 읽어서 자동 계산만 함. 별도 입력 없음.

---

### 종목 및 단위

기존 카테고리/서브타입/단위 체계 그대로.

| 카테고리 | 서브타입 | 단위 |
|---|---|---|
| 달리기 | 러닝 / 트레드밀 | km |
| 사이클 | 실외 / 실내 | km |
| 수영 | — | km |
| 계단 | — | 층 |
| 요가 | — | 분 |
| 복싱 | — | 분 |

---

### progress 집계

```
workouts WHERE
  user_id = $uid
  AND category = $category
  AND sub_type = $sub_type  (null이면 무시)
  AND workout_time BETWEEN $start AND $end
→ SUM(value)
```

### UI 배치

기존 탭(마일리지 / 오늘의 운동) **위에** 조건부 섹션.
챌린지 없으면 섹션 자체 사라짐.

```
┌─────────────────────────────┐
│  클럽 헤더                   │
├─────────────────────────────┤
│  챌린지 섹션 (있을 때만)      │
├─────────────────────────────┤
│ [ 🏆 마일리지 ][ 🏃 오늘의운동]│
└─────────────────────────────┘
```

### 챌린지 카드 UI

**챌린지 1개일 때**: 자동으로 펼쳐진 상태
**챌린지 2개 이상**: 모두 접힌 상태, 탭하면 펼침 (아코디언)

**카드 접힌 상태**
```
[26년 05월 연휴 챌린지!]  5/1~5/7
내 달성률 ████░░ 67%
```

**카드 펼친 상태 (상세)**
```
전체 달성도: 10명 중 7명 달성

─ 내 종목별 달성도 ─
러닝   ████░░  20/30km  67%
수영   ██████  3/3km   100% ✓

─ 참여자 달성도 ─  [검색]
(별명 가나다순)
  강민준  ██████ 100% ✓    ← 탭하면 종목 상세 펼침
    └ 사이클(실외)  100%  ✓

  김철수  ████░░  67%      ← 탭하면 종목 상세 펼침
    └ 러닝  67%

  이영희  ██░░░░  33%
```

> 타인 목표값(숫자)은 비공개. 종목명 + 달성률%만 표시. 본인은 목표값 표시.

### 규칙 확정

| 항목 | 결정 |
|---|---|
| 생성 권한 | 클럽장 + 부매니저 |
| 동시 챌린지 수 | 제한 없음 (카드 복수 표시) |
| 중간 참여 | 가능 |
| 목표 수정 | 불가 |
| 참여 취소 | 불가 |
| 진행 중 챌린지 삭제 | 클럽장/부매니저 가능 |
| 종료 후 노출 유지 | 종료 후 1주일 |
| 탈퇴/강퇴 클럽원 | 데이터 유지, "탈퇴한 회원" 표시 |
| 챌린지 상태 | `active` → `ended` (failed 없음) |
| 아카이브 | 추후 개발 |

## 4. DB 스키마

### `challenges` 테이블 (기존 수정)

- `goal_metric`, `goal_value` → nullable (미사용)
- `status`: `'active' | 'ended'` 로 단순화
- `scope`: `'club'` 고정

### `challenge_participants` 테이블 (신규)

```sql
CREATE TABLE challenge_participants (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  challenge_id  uuid REFERENCES challenges(id) ON DELETE CASCADE,
  user_id       uuid REFERENCES auth.users(id),
  category      text NOT NULL,
  sub_type      text,
  target_value  numeric NOT NULL,
  unit          text NOT NULL,
  created_at    timestamptz DEFAULT now(),
  UNIQUE (challenge_id, user_id)
);
```

## 5. MVP 구현 범위

1. `challenge_participants` 테이블 생성 (마이그레이션)
2. `challenges` 테이블 status/goal 컬럼 정리
3. `challengeService.ts` — 참여자 CRUD + progress 집계 함수
4. 클럽 탭 위 챌린지 섹션 (조건부, 아코디언)
5. 챌린지 생성 모달 (클럽장/부매니저) — 제목 + 기간
6. 참여하기 모달 — 종목 선택 + 목표값 입력
7. 상세 UI — 전체 달성도 + 내 종목별 + 참여자 리스트(검색 포함)
8. 참여자 탭 — 종목별 달성률 펼침
