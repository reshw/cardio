# TODO 260422 — 운동일수 집계 대상 종목 선택 기능

## 배경

현재 클럽 **운동일수**는 `enabled_categories`에 속한 모든 운동을 (마일리지 반영 여부와 무관하게) 집계에 포함한다.

클럽마다 운동일수 집계 기준이 다를 수 있음.
> 예) "요가는 운동일수에서 빼고 싶다", "달리기만 운동일수로 인정"

→ **운동일수에 포함할 종목**을 클럽 어드민이 체크리스트로 직접 선택할 수 있어야 함.

---

## 현재 구조 이해

```
enabled_categories (clubs 테이블)
  └─ 마일리지 계산 + 피드 표시에 사용하는 활성 종목 목록
  └─ [달리기-트레드밀, 달리기-러닝, 요가-일반, ...]

getClubDetailedStats() 집계 시:
  - enabled_categories 안에 있고
  - (countExcludedWorkouts=true OR mileage > 0) 이면
  → workoutDates Set에 추가 → 운동일수 산출
```

`enabled_categories`는 "마일리지/피드 대상 종목"이고,
"운동일수 집계 대상 종목"은 **별도 개념** → 새 컬럼 필요.

---

## 설계

### 새 DB 컬럼 추가
```sql
ALTER TABLE clubs
ADD COLUMN workout_days_categories TEXT[]
DEFAULT NULL;
-- NULL이면 enabled_categories 전체를 그대로 사용 (하위 호환)
```

### 집계 로직 변경
**파일:** `src/services/clubService.ts` → `getClubDetailedStats()`

```typescript
// 운동일수 집계 대상 종목
const workoutDaysCategories: string[] =
  club.workout_days_categories ?? enabledCategories;

// 운동일수 집계 조건 변경
if (workoutDate && workoutDaysCategories.includes(record.category)) {
  userStatsMap[record.user_id].workoutDates.add(workoutDate);
}
```

→ 기존 `countExcludedWorkouts` 로직은 제거하거나 병행 검토

### Club 인터페이스 추가
**파일:** `src/services/clubService.ts`
```typescript
export interface Club {
  // 기존...
  workout_days_categories?: string[] | null; // null = enabled_categories 전체 사용
}
```

### updateClub() 파라미터 추가
```typescript
data: {
  // 기존...
  workout_days_categories?: string[] | null;
}
```
