# 수정사항 260413

## 1. 운동 기록 단위 저장 버그 수정

**문제**
- 계단 → 층수입력 선택 후 저장하면 UI에는 "층"으로 표시되지만 DB에 "분"으로 저장됨
- 피드 카드에서 단위가 "분"으로 잘못 표시됨

**원인**
- `AddWorkout.tsx` 저장 시 `selectedCategory!.unit` (카테고리 메인 단위) 사용
- 반면 UI 표시는 `displayUnit` (선택된 서브타입 단위) 사용 → 표시와 저장 불일치
- `AddWorkoutModal.tsx`는 이미 `displayUnit` 사용 중이라 정상이었음

**수정**
- `AddWorkout.tsx` 저장 시 `unit: selectedCategory!.unit` → `unit: displayUnit as WorkoutUnit`

**변경 파일**
- `src/pages/AddWorkout.tsx`

## 2. 운동 종목 단위 구조 개선 (어드민)

**문제**
- 운동 종목(workout_types)에 카테고리 자체 단위(`unit`)와 각 서브타입 단위가 독립적으로 관리되어 충돌 가능
- 서브타입이 있는 종목에서 카테고리 단위는 실질적으로 사용되지 않아야 하는데 어드민 UI에서 그 관계가 불명확했음

**수정**
- 어드민에서 운동 종목 추가/수정 시, 서브타입이 1개 이상 있으면 카테고리 단위 필드를 읽기전용으로 전환
  - "자동 — 첫 번째 세부타입 기준" 문구 표시
- 서브타입 추가 시 → 카테고리 단위를 첫 번째 서브타입 단위로 자동 동기화
- 서브타입 제거 시 → 남은 첫 번째 서브타입 단위로 재동기화 (서브타입이 모두 없어지면 기존 값 유지)

**변경 파일**
- `src/pages/AdminWorkoutTypes.tsx`

## 3. WorkoutUnit 타입 확장

**문제**
- `workoutService.ts`의 `WorkoutUnit` 타입에 `'회'`, `'세트'`가 누락
- `workoutTypeService.ts`에서는 이미 지원 중이라 불일치 상태

**수정**
- `WorkoutUnit = 'km' | 'm' | '층' | '분'` → `'km' | 'm' | '층' | '분' | '회' | '세트'`

**변경 파일**
- `src/services/workoutService.ts`
