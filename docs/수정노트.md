## 2. 클럽 상세 통계 모달 — 선택 월 연동 + UTC+9 운동일수 버그 수정

### 2-1. 선택 월 연동

**파일**: `src/pages/Club.tsx`

#### 문제
"운동별 상세 통계" 모달이 항상 `new Date()`로 현재 월 고정.
클럽 화면에서 이전 달로 이동해도 모달은 항상 이번 달을 표시.

#### 변경 내용
- 모달에 하드코딩된 `new Date().getFullYear()` / `new Date().getMonth() + 1` 제거
- 클럽 화면의 기존 `selectedMonth` state를 그대로 전달

---

### 2-2. 운동일수(workout_days) 항상 0 버그 수정

**파일**: `src/services/clubService.ts`, `src/services/workoutService.ts`

#### 원인 (버그 3개)

| # | 위치 | 문제 |
|---|------|------|
| 1 | `clubService.saveWorkoutMileage()` | `workout_date` 필드가 upsert에 **누락** → `workoutDates.add()` 미실행 → 운동일수 항상 0 |
| 2 | `clubService.saveWorkoutMileage()` | year/month를 `created_at`(서버 UTC 자동생성) 기준 계산 → KST 자정 전후 운동이 **다른 월**로 저장될 수 있음 |
| 3 | `clubService.saveWorkoutMileage()` | `workout_date`를 `.toISOString().split('T')[0]`(UTC 날짜) 저장 → KST 기준 전날 날짜로 기록될 수 있음 |

#### 변경 내용
- `saveWorkoutMileage()` 파라미터 `created_at` → `workout_time`으로 교체
- year/month/workout_date 모두 `workout_time` 기준 **로컬(KST) 날짜**로 계산
- `workout_date` upsert에 추가

```ts
// 변경 전
const workoutDate = new Date(workout.created_at);
// workout_date 없음

// 변경 후
const d = new Date(workout.workout_time);
const year = d.getFullYear();
const month = d.getMonth() + 1;
const workout_date = `${year}-${String(month).padStart(2, '0')}-${String(d.getDate()).padStart(2, '0')}`;
// upsert에 workout_date 포함
```

- `workoutService.ts` 두 호출부(운동 추가, 운동 수정) 모두 `workout_time` 전달로 변경

#### 주의
기존에 저장된 `club_workout_mileage` 레코드 중 `workout_date = null`인 것들은 운동일수 집계 여전히 0.
클럽별 마일리지 재계산을 실행하면 수정됨.

## 3. 클럽 피드 — "n번째 운동" 기준 created_at으로 변경

**파일**: `src/services/clubService.ts`, `src/services/feedService.ts`

### 문제
카톡 공유 시 표시되는 "오늘 클럽 n번째" 문구가 `workout_time`(운동한 시각) 기준 정렬 순서였음.
- 저녁에 오전 7시 운동을 등록하면 피드 상단에 삽입되면서 기존 번호가 밀림
- 이미 30번째까지 공유됐는데 갑자기 12번째가 등장하는 상황 발생
- 유저가 컨트롤할 수 없는 타인의 `workout_time` 값에 의해 순번이 바뀌는 구조

### 변경 내용
- **피드 표시 순서**: `workout_time` 기준 유지 (직관적 시간순 정렬)
- **n번째 번호**: `created_at`(등록 시각) 기준으로 분리

#### clubService.ts
fetch 후 `created_at` 기준으로 별도 순번 맵 생성, `workout_number`에 할당:
```ts
const createdAtOrder = [...workouts]
  .sort((a, b) => new Date(a.created_at).getTime() - new Date(b.created_at).getTime());
const workoutNumberMap = new Map(createdAtOrder.map((w, i) => [w.id, i + 1]));
```

#### feedService.ts (`getTodayWorkoutNumber`)
날짜 범위는 `workout_time` 기준 유지, 정렬만 `created_at` 으로 변경:
```ts
.order('created_at', { ascending: true })  // 변경 전: workout_time
```

### 결과
한번 등록된 운동의 "n번째" 번호는 이후 다른 사람이 과거 시간으로 운동을 올려도 변하지 않음.

## 4. 클럽 랭킹 — "일반 회원" 필터 탭 구분 표시 추가

**파일**: `src/pages/Club.tsx`, `src/App.css`

### 배경
클럽 랭킹에는 "전체 / 일반 회원 / 명예의 전당" 필터 탭이 있음.
전체 탭과 일반 회원 탭을 각각 스크린샷 찍어 공유할 때, 화면만 보면 어느 탭인지 구분이 안 되는 문제가 있었음.

### 변경 내용
- 일반 회원 탭 선택 시 마일리지 숫자 바로 아래에 `(일반)` 텍스트 표시
- 등수 뱃지 옆이 아닌 마일리지 합산값 아래 우측 정렬로 배치 (작은 보조 글씨)

### 결과
전체 탭 스크린샷과 일반 회원 탭 스크린샷을 나란히 놓았을 때 `(일반)` 유무로 즉시 구분 가능.

## 5. 클럽 랭킹 — 리더보드 표시 범위 개선

**파일**: `src/pages/Club.tsx`

### 변경 전
- 항상 Top 10 고정 표시
- 내 순위가 10위 밖이면 내 순위 ±1명 추가 (총 최대 13명)
- 10위 권 밖일때 중략, 후략 발생 

### 변경 후
- **내 순위 20위 이내**: 1~20위 전체 표시
- **내 순위 20위 밖**: Top 5 + 내 순위 기준 ±3명 (총 최대 12명)
- 구간 사이에 **중략** (`...`) 표시, 20위 밖 구간은 **후략** 처리

### 의도
순위권 안에 있는 유저는 전체 경쟁 상황을 한눈에 파악.
순위권 밖 유저는 스크롤을 많이 내려야 본인 순위를 확인할 수 있는 UX 불편을 해소하기 위해, Top 5 + 내 주변 순위만 간결하게 모아 표시. 동기부여는 유지하되 스크롤 피로 최소화.
