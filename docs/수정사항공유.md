# 앱 수정사항 공유

> 운영진 공유용 문서입니다.
> 기술적인 코드나 변수 설명은 생략하고, 무엇이 왜 바뀌었는지 중심으로 작성했습니다.

---

## 1. 기록 탭 — 통계를 월별로 볼 수 있게 됩니다

### 기존
내 운동 기록의 통계를 보면 항상 "이번 달"만 표시됐습니다.

### 변경
상단에 `‹ 2026년 4월 ›` 형태의 월 이동 버튼이 생겼습니다.
가운데 년/월을 누르면 달력처럼 월과 연도를 직접 골라서 볼 수 있습니다.
과거 어느 달이든 그 달의 운동 통계를 바로 확인할 수 있습니다.

### 왜 바꿨나요?
이번 달만 보이는 건 너무 제한적이었습니다. 지난달 얼마나 했는지, 특정 달에 얼마나 뛰었는지 보고 싶어도 볼 수가 없었거든요.

---

## 2. 클럽 상세 통계 — 선택한 달에 맞게 표시

### 기존
클럽 랭킹에서 월을 이전 달로 이동해 놓고 상세 통계를 열어도, 안에서는 항상 이번 달 데이터가 표시됐습니다.

### 변경
이제 클럽 화면에서 선택한 달과 상세 통계 화면이 일치합니다.
3월로 이동해서 상세 통계를 열면 3월 데이터가 나옵니다.

---

## 3. 클럽 상세 통계 — 운동일수가 항상 0으로 나오는 버그 수정

### 기존
운동별 상세 통계 표에서 "운동일수" 항목이 항상 0으로 표시됐습니다.

### 원인
앱은 운동 데이터를 저장할 때 두 가지 시간을 기록합니다.
- **운동한 시각** — 유저가 직접 입력하는 실제 운동 시간
- **등록한 시각** — 앱에 올린 순간 자동으로 찍히는 시간

운동일수를 세는 부분에서, 한국 시간이 아닌 세계 표준시 기준으로 날짜가 저장되다 보니 자정 전후에 등록된 운동의 날짜가 하루씩 어긋나는 문제가 있었습니다. 거기에 날짜 자체가 아예 저장되지 않는 버그도 겹쳐 있었습니다.

### 변경
날짜를 한국 시간 기준으로 올바르게 저장하도록 수정했습니다.
앞으로 새로 등록하는 운동부터 운동일수가 정상 집계됩니다.

> 이전에 등록된 기록들은 재집계가 필요할 수 있습니다. 클럽메뉴에서 마일리지 재계산 하면, 다시 계산해옵니다.

---

## 4. 클럽 피드 — "오늘 n번째" 표시 기준 변경

### 기존
카톡으로 공유할 때 붙는 "오늘 클럽 n번째" 문구가 운동한 시각 순서로 매겨졌습니다.
그러다 보니 저녁에 오전 7시 운동을 뒤늦게 올리면, 기존에 공유된 번호들이 밀리거나 바뀌는 현상이 생겼습니다.

예: 이미 30번째까지 공유됐는데 갑자기 12번째가 새로 등장하는 상황

### 변경
이제 "n번째"는 앱에 **등록한 순서** 기준으로 매겨집니다.
한번 "17번째"로 공유된 운동은, 이후 누군가 과거 시간으로 운동을 올려도 계속 17번째로 유지됩니다.

### 왜 이게 맞나요?
"오늘 클럽 n번째 운동!"은 몇 시에 운동했느냐가 아니라, 몇 번째로 **올렸느냐**의 개념이 더 자연스럽습니다. 어쩔 수 없는 타인의 운동 시각 때문에 내 번호가 바뀌는 건 이상하니까요.

## 5. 클럽 랭킹 — "(일반)" 표시 추가

### 기존
랭킹을 "전체"로 볼 때와 "일반 회원"으로 필터링할 때, 화면만 봐서는 어느 탭인지 구분이 어려웠습니다.
스크린샷을 찍어서 공유할 때 특히 헷갈렸습니다.

### 변경
"일반 회원" 탭을 선택하면 마일리지 숫자 아래에 작게 **(일반)** 이 표시됩니다.
전체 탭과 일반 탭 스크린샷을 나란히 놓으면 바로 구분할 수 있습니다.

## 6. 클럽 랭킹 — 내 순위 보이는 범위 개선

### 기존
항상 상위 10명만 표시하고, 내 순위가 10위 밖이면 나와 바로 위아래 1명씩만 추가로 보였습니다.
최초의 설계의도는 하위권일수록 스크롤을 많이 내려야 내 순위를 확인할 수 있으니, 중략을 하는 의도였습니다.

### 변경
- **내 순위가 20위 이내** — 1위부터 20위까지 전체를 한눈에 표시
- **내 순위가 20위 밖** — 상위 5명 + 내 주변 위아래 3명씩 표시
- 중간 생략된 구간은 "..." 으로 표시해서 어디의 몇명을 생략했는지 알 수 있습니다.

### 왜 바꿨나요?
순위권 안에 있는 분들은 경쟁 상황 전체를 보고 싶고, 순위권 밖인 분들은 스크롤 없이 내 위치를 바로 파악하고 싶으실 겁니다. 두 가지를 동시에 배려하되, 볼수있는 범위를 넓히는 방식으로 개선했습니다.
